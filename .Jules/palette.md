## 2024-05-29 - Accessible Animated Dropdowns with Framer Motion
**Learning:** When building custom animated dropdown menus (like the `ThemeSwitcher` using Framer Motion's `AnimatePresence`), standard semantic HTML menus lose their native accessibility features. To replicate native behavior, the wrapper needs `role="menu"`, interactive elements need `role="menuitem"`, headers/spacers need `role="presentation"`, and the trigger button requires `aria-haspopup="menu"` alongside a dynamically updated `aria-expanded={isOpen}`. Furthermore, custom JS handlers for the Escape key and outside-click events are mandatory for an inclusive user experience.
**Action:** Always pair `AnimatePresence` driven popovers or menus with a robust `useEffect` hook handling `Escape` and outside clicks, and explicitly map ARIA roles to compensate for the loss of native `<select>` or `<details>` element accessibility.
