## 2025-05-18 - Animated Dropdowns and Wrapper Components Accessibility
**Learning:** When building custom animated dropdowns (e.g., `ThemeSwitcher` with Framer Motion), `role="menu"`, `role="menuitem"`, `aria-haspopup`, and `aria-expanded` are essential to ensure proper screen reader support. Non-interactive elements within the menu must receive `role="presentation"`. Additionally, when creating reusable interactive wrapper components (like `MagneticButton`), unconditionally wrapping children in `<button>` tags can cause invalid HTML nesting (e.g., `<a>` inside `<button>`) if the component is used as a purely visual wrapper.
**Action:** Always add proper ARIA roles to custom dropdown menus. For wrapper components, conditionally render the root element (e.g., return a fragment `<>{content}</>` if neither `href` nor `onClick` is provided) to maintain semantic and valid HTML.
