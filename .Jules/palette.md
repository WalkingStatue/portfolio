## 2024-05-08 - Accessible Custom Animated Dropdowns
**Learning:** Custom animated dropdowns (like those built with Framer Motion) often lack the semantic HTML structure of native `<select>` elements, making them opaque to screen readers. We discovered that without specific roles (`menu`, `menuitem`, `presentation`) and states (`aria-expanded`, `aria-haspopup`), users relying on assistive technologies cannot perceive the dropdown's interactive state or navigate its options. Furthermore, standard keyboard navigation expectations—like using the Escape key to close a dropdown—must be manually re-implemented when replacing native elements.
**Action:** When building or modifying custom interactive dropdowns, always ensure the trigger element has `aria-expanded` and `aria-haspopup="menu"`. The container must use `role="menu"`, individual options `role="menuitem"`, and non-interactive structural elements `role="presentation"`. Crucially, always attach an event listener to the `Escape` key to close the dropdown and return focus to the trigger.
