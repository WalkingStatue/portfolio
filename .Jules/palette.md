## 2026-04-03 - Accessible Framer Motion Dropdowns
**Learning:** Animated dropdowns built with Framer Motion `motion.div` lack semantic meaning for screen readers by default. When wrapping a list of buttons in a motion element to create a custom menu, it is critical to explicitly add `role="menu"` to the container, `role="menuitem"` to the interactive children, and `role="presentation"` to any decorative/structural headers to ensure proper screen reader navigation. Furthermore, the trigger button must include `aria-haspopup="menu"` and `aria-expanded={isOpen}`.
**Action:** Always verify that custom animated UI components constructed from generic `motion.div` containers receive the appropriate ARIA roles corresponding to their functional pattern (e.g., menu, dialog, tablist).
