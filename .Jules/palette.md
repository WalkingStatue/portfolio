## 2024-05-15 - [ThemeSwitcher Dropdown Accessibility & State Init]
**Learning:** For custom animated dropdowns (like ThemeSwitcher built with Framer Motion), standard button clicks lack context for screen readers. Additionally, reading from `localStorage` within `useEffect` to initialize state leads to ESLint cascading render warnings and Flash of Unstyled Content (FOUC). Using `document.documentElement.className = ''` can cause immutability lint errors.
**Action:** Used lazy initialization for `useState` to pull from `localStorage` immediately upon rendering. Switched to `document.documentElement.setAttribute('class', ...)` for safer DOM class management. Added `aria-haspopup="menu"`, `aria-expanded`, `role="menu"`, `role="presentation"`, and `role="menuitem"` to ensure the custom dropdown functions accessibly.
