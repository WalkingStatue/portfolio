## 2024-05-26 - Accessible ThemeSwitcher and MagneticButton HTML nesting
**Learning:** For animated custom dropdowns, applying explicit ARIA roles (`menu`, `menuitem`, `presentation`) and expanding accessibility beyond keyboard navigation (closing on click outside/escape key) is essential. Moreover, using interactive wrapper components like `MagneticButton` can inadvertently result in invalid HTML (such as `<a>` inside `<button>`) if default wrapper elements are hardcoded instead of being rendered conditionally based on provided props.
**Action:** When building interactive dropdowns, always ensure they are accessible (`aria-expanded`, focus trapping, role definitions, escape key listeners). For generic UI wrapper components, conditionally render semantically valid base elements (such as `Fragment` vs `button` vs `a`) based on whether interaction props (`onClick` or `href`) are provided.
