## 2025-05-12 - Interactive element structure and keyboard navigation
**Learning:** For custom dropdown elements like the Theme Switcher, missing basic interactive paradigms (closing on 'Escape' or 'click-outside') breaks expected UX heavily. In addition, conditional wrapper components (like `MagneticButton`) can unintentionally create invalid DOM layouts (e.g. `<a>` inside a `<button>`) leading to accessibility issues, or lack robust regex checks for `target="_blank"`.
**Action:** When building custom interactive components such as dropdowns, consistently implement `Escape` and 'click-outside' listener hooks. Additionally, strictly add `role="menu"`, `role="menuitem"`, `aria-haspopup="true"`, and `aria-expanded={isOpen}`. Always ensure conditional root wrapper components gracefully fallback to `<>{content}</>` if interactive properties (`onClick` / `href`) are absent, avoiding generic `<button>` wrappers.
