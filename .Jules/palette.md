## 2024-05-18 - Animated Dropdown Menu Accessibility
**Learning:** Custom animated dropdowns (like with Framer Motion) require explicit ARIA attributes since they aren't native `<select>` or `<menu>` elements. It is crucial to add `aria-haspopup="menu"` and `aria-expanded` to the trigger, `role="menu"` to the dropdown container, `role="menuitem"` to interactable items, and `role="presentation"` to non-interactive layout containers within the menu to prevent screen readers from announcing them as interactive. Furthermore, standard keyboard navigation (closing via the Escape key) must be manually implemented to maintain keyboard accessibility.
**Action:** When building custom animated dropdown menus, always add `role="menu"`, `role="menuitem"`, `aria-haspopup`, and `aria-expanded` attributes. Also ensure non-interactive elements within the menu have `role="presentation"`, and always support closing the menu via the Escape key.
