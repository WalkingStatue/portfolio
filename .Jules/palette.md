## 2024-05-16 - Custom Animated Dropdown Accessibility and Keyboard Behavior
**Learning:** When building custom animated dropdown menus (like those using Framer Motion) that mimic native `<select>` or `<details>` behavior, they lack built-in screen reader semantics and keyboard event handling. Specifically, failing to close on `Escape` or outside clicks creates keyboard traps and frustrating UX, while missing `role="menu"` and `role="menuitem"` leaves screen reader users without context about the available options or the component's state.
**Action:** Always manually add `aria-expanded`, `aria-haspopup`, `role="menu"`, `role="presentation"`, and `role="menuitem"` to custom dropdown components. Furthermore, attach document-level `keydown` (for `Escape`) and `mousedown` (for outside clicks) event listeners while the menu is open to ensure standard expected behavior and prevent keyboard trapping.
