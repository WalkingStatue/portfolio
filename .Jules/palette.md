## 2026-05-19 - Accessible and Resilient Animated Dropdowns
**Learning:** When building custom animated dropdowns (e.g., using Framer Motion) without standard HTML `<select>` elements, screen reader accessibility and intuitive UX (like dismissing via outside clicks or the Escape key) are frequently overlooked. Relying solely on `className` toggles for theming can also trigger React immutability warnings if modified directly on `document.documentElement`.
**Action:** Always map custom dropdowns to standard ARIA roles (`menu`, `menuitem`, `presentation`) and manage `aria-haspopup`/`aria-expanded` states. Implement `mousedown` and `keydown` event listeners to support expected UI behaviors (outside click and Escape key dismissal). When interacting with global DOM nodes for theming, use `.setAttribute("class", ...)` to bypass React immutability conflicts while maintaining clean global state.
