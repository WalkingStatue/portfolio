## 2024-04-11 - [Accessibility for Framer Motion Dropdowns & DOM Class Mutation]
**Learning:** For custom animated dropdowns (like those built with Framer Motion), standard screen reader compatibility is often missing unless explicitly defined. Essential attributes include `aria-expanded`, `aria-haspopup`, `role="menu"`, `role="presentation"`, and `role="menuitem"`. Additionally, when toggling themes via class updates on the `document.documentElement`, assigning a string directly to `className` or doing an empty string assignment triggers lint/DOM errors. It's best to use `setAttribute('class', ...)` for multi-class strings and avoiding immutability lint errors in React.
**Action:** When building dropdown components, always explicitly apply `role="menu"` to the container, `role="menuitem"` to interactable options, and `role="presentation"` to static headers/separators. Also, use `setAttribute('class', ...)` instead of `className` when overriding root classes to prevent `DOMException` and linter issues.
