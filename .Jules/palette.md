## 2024-05-15 - [Execution Mode Dropdown Accessibility]
**Learning:** When building custom animated dropdowns (e.g., with Framer Motion), it's crucial to explicitly manage screen reader accessibility by adding `role="menu"`, `role="menuitem"`, `aria-haspopup`, and `aria-expanded`. Non-interactive elements within the menu must receive `role="presentation"`. Additionally, keyboard navigation needs manual management, such as adding an `Escape` key listener to close the dropdown.
**Action:** Always verify keyboard accessibility (tabbing and escaping) and use ARIA roles for any custom dropdown component that overrides native select/menu behavior. When overriding global themes dynamically via document classes in React, use `setAttribute('class', ...)` instead of mutating `.className` directly to satisfy immutability rules.
