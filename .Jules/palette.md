## 2024-04-03 - ThemeSwitcher Accessibility and FOUC Prevention
**Learning:** Found that custom dropdown menus lacking `role="menu"` and `aria-haspopup` limit screen reader usability, and initializing state without lazy-loading `localStorage` can cause a flash of unstyled content (FOUC). Also noted that directly assigning `document.documentElement.className = ''` followed by `classList.add(...)` triggers React immutability/side-effect lint warnings if not properly managed, and it is better to use `setAttribute('class', ...)` conditionally. Additionally, empty string fallbacks (`themeObj.class || ''`) are critical when updating the class attribute dynamically, to avoid applying a literal `"undefined"` class to the HTML tag.
**Action:** Always verify custom interactive dropdown components receive `aria-haspopup`, `aria-expanded`, and the appropriate ARIA roles (`menu`, `menuitem`, `presentation`). When persisting theme state from `localStorage`, strictly use lazy state initialization (`useState(() => localStorage.getItem('key'))`) and only perform document-level side-effects (e.g. `setAttribute('class', ...)`) within a `useEffect` layout hook to prevent cascading render cycles and maintain lint compliance.
