## 2024-05-21 - Custom Animated Dropdown Accessibility
**Learning:** When building custom animated dropdown menus (like the Framer Motion `ThemeSwitcher`), native HTML `<select>` accessibility is lost. Critical attributes (`role="menu"`, `role="menuitem"`, `aria-haspopup`, `aria-expanded`) are frequently omitted. Furthermore, non-interactive elements within the menu must be marked with `role="presentation"` to prevent screen reader confusion. Keyboard interactions (like Escape to close) and click-outside handling are also necessary to mirror native behavior and ensure full keyboard navigation.
**Action:** Always verify custom dropdowns for `role="menu"`, `aria-haspopup`, `aria-expanded`, and Escape key closure support. Document this pattern to consistently apply it to all interactive overlay components.
