## 2024-05-15 - Interactive Wrapper Elements and Dropdown Accessibility
**Learning:** Wrapper components like `MagneticButton` can inadvertently create invalid HTML (like `<a>` tags inside `<button>`) when they default to wrapping children in a `<button>` tag if an `href` isn't provided but no `onClick` is handled either. Custom animated dropdowns (like `ThemeSwitcher`) need strict ARIA roles (`menu`, `menuitem`, `presentation`) to maintain accessibility, as screen readers cannot interpret visually styled `div` components.
**Action:** Always conditionally render semantic wrapper elements based on provided props (e.g., return a fragment if neither `href` nor `onClick` exists). Apply explicit `role="menu"` and related ARIA attributes when building interactive custom select/dropdown menus.
